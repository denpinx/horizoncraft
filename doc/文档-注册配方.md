# 注册配方

#### 详细请查看[Json Schema](../config/schema/recipe.json)

## 打开“config\recipe\”目录,往里面添加任意文件，文件名不限,但是最好还是根据配方类型进行分类在不同的文件夹中比较好

## 列1 定义合成配方:
```Json
    {
  "type": "craft",
  "tag": "player",
  "recipes": [
    {
      "template": [
        "#"
      ],
      "mask": {
        "#": "oak_log"
      },
      "result": [
        "oak_plank",
        4
      ]
    },
    {
      "template": [
        "###",
        "-!-",
        "-!-"
      ],
      "mask-tag": {
        "-": "air",
        "!": "stick",
        "#": "plank"
      },
      "result": [
        "wood_pickaxe",
        1
      ]
    }]
    }
```

## 列2 定义处理配方:

```Json
{
  "type": "process",
  "tag": "furnace",
  "recipes": [
    {
      "process": 20,
      "cost": [
        [
          "grass",
          1
        ]
      ],
      "result": [
        [
          "dirt",
          1
        ]
      ]
    },
    {
      "process": 20,
      "cost": [
        [
          "dirt",
          1
        ]
      ],
      "result": [
        [
          "grass",
          1
        ]
      ]
    }
  ]
}
```

# 解析

- template的每个string必须对其长度，以最长的为准
- mask-tag使用的是词典匹配，可以在配置物品的时候配置物品的词典，如果没有配置，则会自动以物品名称生成，表明这是一个单独的物品。
- mask则是完整匹配物品名，不管词典名
- result只能是第一个是物品名，第二个是物品数量，且物品名只能是确定的物品名，不适配自动匹配。
- mask必须把air显示标明，表示这个格位必须为空
