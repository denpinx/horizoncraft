# 注册方块

#### 详细请查看[Json Schema](../config/schema/block.json)

## 打开“config\block\”目录,往里面添加任意文件，文件名不限,但是最好还是根据方块类型进行分类在不同的文件夹中比较好

## 列:
```Json
    {
  "方块名1": {},
  "方块名2": {
    "collide": false,
    "cube": false,
    "tiletype": "tile",
    "components": {
      "TickComponent": {
        "Name": "BottomCheck",
        "Max": 20
      }
    },
    "loot": [
      {
        "name": "方块名1",
        "drop-chance": 1,
        "amount-chance": [
          {
            "amount": 1,
            "chance": 1
          }
        ]
      }
    ],
    "tags": {
      "type": "shovel"
    },
    "state": {
      "状态一": {
      },
      "状态二": {
        "texture": "oak_log_状态2"
      }
    }
  }
}
```
# 解析

- 绝大多数字段可以不写,会有默认值
- 物品可以多个组件,但是同一个组件类只能有一个，不能完全重复，但可以是派生的子类或父类。
- state属性的 状态名 其实无所谓，只是用来定义用的，本质上数组。
- 物品配置的优先级 > 方块配置,会覆盖方块的配置，方块会默认自动生成对应的物品配置(如果没有被提前加载的话)
