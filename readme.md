# 基于Godot和C#开发的类Minecraft2D游戏《HorizonCraft》

项目已经升级到 godot 4.7

目前已添加55种方块，80种物品,基本上都有对应的贴图。

## 目前支持的功能

- 无限地图
- 死亡重生
- 白昼变化
- 方块光照
- 物品合成
- 结构生成
- 生物群系
- 物品使用
- 多人联机
- 服务端导出
- 物品管理器
- 内置建筑编辑器
- 基于组件和系统驱动的方块

## 白昼变化

- 天空光照值会随着时间而变化
- 天空在不同时间段有不同的颜色变化

## 无限地图

- 基于sqlite存储和查询区块的二进制数据
- 区块化加载/卸载
- 玩家周围 3*3的区块将会被标记成强加载
- 玩家周围 5*5以内，3*3以外的区块将会被标记成弱加载
- 超出5*5范围的，将会被进入卸载计数

## 实体

- 实体会跟随区块的加载而加载，卸载而卸载
- 如果实体所在的区块没有加载，将会发起区块加载的任务，直到区块被加载

## 组件和系统驱动的方块,物品,实体
    
- 构建相同功能的变种
- 搭配不同的系统和组件来构建新对象

## 物品管理器

- 创造模式下可一键获取物品
- 生存模式下可查询物品配方

## 多人联机

- 并行区块io优化
- 并行区块Tick优化
- 主机端压力减少，系统会把实体的所属权转移到加载实体所在区块玩家上，让玩家去更新实体的物理属性，并更新回服务端，这样主机就不用加载全部区块的TileMap了

## 服务端导出

- 可以导出为专用服务端

## 世界模式
    
- 单机模式
- 主机模式
- 客户端模式
- 预览模式（主菜单的背景其实就是预览模式下的世界）

## 结构生成

- 结构可以跨区快生成（其实是直接算出来的，不用管周围区块有没有加载）
- 结构大小最大只能为区块大小

## 备注

- F2 创造模式/生存模式切换
- F3 调试信息
- F4 查看区块信息

## 环境要求

- Godot Engine 4.7 (.NET)
- .NET SDK 8.0

## 运行环境

- Windows 10/11

## 运行项目

- 下载并解压该项目（或使用git clone）
- 运行Godot编辑器，打开该项目
- 点击编辑器右上角的运行
