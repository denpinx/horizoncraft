# 最近更新

    1.实现多人模式下，多个玩家分布再不同区域时，区块会自动对区块集进行连续分组，对分组的区块用并行处理时刻更新，解决了单线程下,时刻耗时和区块加载数量成直线关系的问题,也就是异步区块更新。
    2.修复TileMapLayer其实是的动态更新问题，现在为被动更新,只有在区块数值发生变化时重新更新区块,现在帧率稳定500-1100帧。
    3.修复了用于创建组件的Lambda对象构造表达式实际上是每次运行时用闭包重新用表达式构建Lambda并运行的问题,现在区块生成的性能从35ms每个区块提升到了8ms每个,每个组件的构建速度提升1825倍(或则说原本就写的很差..),显著减少运行时的最大处理耗时,36ms-> 24ms
    4.之后的一些方块和物品的aseprite贴图源文件在贴图导出后会被我保存到"texture_source"文件夹内,方便重新编辑。
    5.修复实体掉落出未加载区块时不会被卸载，现在当实体所在的区块未加载时，会被加入到区块加载队列，会被异步加载，并在20tick内没有被玩家加载时自动卸载。
    6.修复实体同步问题,目前又可以正常同步了。
    7.实现多语言支持,可以在运行时切换语言。
    8.物品管理器添加搜索，翻页和双向配方查询功能。
    9.矿物生成结构改进，现在矿脉更聚集和连续。    
    10.生物群系的滑块权重生成规则改变，现在会先放大32倍生物群系的权重列表，然后再反复随机交换列表顺序，最后重新生成生物群系的权重，现在生成的生物群系在小尺度下是不会出现重复的规则了，每个世界的规则都完全不一样。原本的生成权重顺序完全就是根据注册顺序决定的，现在完全随机且可控化了。
    11.多人聊天已完成,可以同步了

# 多人相关

    目前已经客户端实现增量更新至服务端，服务端增量同步给客户端。
    客户端代理服务端计算实体物理，并在服务端加载客户端的代理实体所在区块时拿回所属权。

### 不要直接在玩家节点中直接调用使世界服务等各种功能服务实现事件,会导致结果只会在客户端生效

    因为玩家节点的一切操作都是客户端操作，玩家节点内的输入操作只是决定什么时候调用该事件。
    必须先在 PlayerServiceBase.PlayerEvents类中 创建对应的事件方法。
    然后在PlayerServiceNode类型中实现该方法的远程调用版本。
    最后在ClientPlayerEvents类中override该方法,将其改成rpc远程调用。

    总结: 客户端PlayerNode调用->ClientPlayerEvents->调用服务端PlayerServiceNode调用服务端->PlayerEvents事件结果返回->客户端PlayerServiceNode,直接在这里处理,不要再调用PlayerEvents处理，回导致无限回调。
